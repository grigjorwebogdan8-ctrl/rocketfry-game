## 🎯 О проекте

**Crash Game** — это полностью функциональное казино-приложение с:

- ✅ **Игровой механикой**: раунды с обратным отсчетом, полетом ракеты и крашем
- ✅ **Системой ставок**: ввод ставки, быстрые кнопки (+1, +5, +10), валидация баланса
- ✅ **Живым онлайном**: имитация других игроков с автокэшаутом
- ✅ **Анимациями**: звездный фон с параллаксом, плавная траектория ракеты
- ✅ **Навигацией**: вкладки "Краш", "Рулетка", "Профиль"

**Демо**: готовое приложение работает в Figma Make без возможности прямого скачивания.

## ❓ FAQ

### 1. Где скачать готовый проект?

В текущей версии Figma Make **нет прямой кнопки экспорта**. Вам нужно:
- Воссоздать проект вручную по документации
- Связаться с поддержкой Figma для получения актуальной информации

### 2. Какие файлы критически важны?

- `src/styles/theme.css` — токены Tailwind v4
- `src/app/App.tsx` — главный компонент
- `src/app/lib/utils.ts` — функция `cn()`
- `package.json` — зависимости

### 3. Можно ли запустить проект без TypeScript?

Да, но придется:
1. Переименовать все `.tsx` → `.jsx`
2. Удалить все типы (`: string`, `interface`, и т.д.)
3. Изменить `vite.config.ts` → `vite.config.js`

### 4. Почему не работают стили?

Проверьте:
1. Установлен ли `@tailwindcss/vite`
2. Скопирован ли файл `src/styles/theme.css` полностью
3. Правильно ли импортированы стили в `src/styles/index.css`

### 5. Как добавить localStorage для сохранения баланса?

Добавьте в `App.tsx`:

```tsx
useEffect(() => {
  const saved = localStorage.getItem('balance');
  if (saved) setBalance(parseFloat(saved));
}, []);

useEffect(() => {
  localStorage.setItem('balance', balance.toString());
}, [balance]);
```

### 6. Можно ли добавить backend?

Да! Вам нужно:
1. Создать REST API или WebSocket сервер
2. Заменить `generateCrashPoint()` на запрос к серверу
3. Добавить авторизацию пользователей
4. Сохранять ставки и баланс в базе данных
